 Error Handling Practice Routes

```http
GET /reviews/:id
→ Return 404 if review not found

POST /reviews
→ Return 400 if fields missing

PATCH /bookings/:id
→ Return validation error

DELETE /conflicts/:id
→ Return proper delete response

GET /admin/reviews
→ Return 401 if unauthorized
```

 Request Validation Practice

```http
POST /reviews
→ Validate rating between 1-5

POST /bookings
→ Validate booking date

POST /auth/register
→ Validate email/password

POST /conflicts

```
